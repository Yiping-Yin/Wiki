# Reflexion & Self-Improvement

Reflexion is a framework where a language model agent improves its own behavior across trials by generating **verbal self-critiques** and storing them in an episodic memory, rather than by updating weights. Introduced by Shinn, Cassano, Berman, Gopinath, Narasimhan, and Yao in March 2023 ("Reflexion: Language Agents with Verbal Reinforcement Learning"), it reframes reinforcement learning for LLMs: instead of gradient updates from a scalar reward, the agent receives a reward signal, *reflects in natural language* on what went wrong, and prepends that reflection to its next attempt's context. The weights never change — the policy improves because the prompt gets smarter.

The technique caught on because it sidesteps the cost and instability of RLHF/PPO for agentic tasks while delivering large gains: 91% pass@1 on HumanEval (vs. 80% for GPT-4 baseline at the time), and substantial lifts on ALFWorld and HotpotQA. Today the pattern lives on in virtually every serious agent stack — SWE-agent, Devin-style coders, Voyager, and most "self-correcting" coding loops are descendants of Reflexion's actor / evaluator / self-reflection triad.

## Key idea

Standard RL learns a parameterized policy $\pi_\theta(a \mid s)$ by maximizing $\mathbb{E}[\sum_t r_t]$ via gradients on $\theta$. Reflexion freezes $\theta$ and instead learns a **textual memory** $\text{mem}_t$ that conditions the policy:

$$
\pi(a_t \mid s_t, \text{mem}_t), \qquad \text{mem}_{t+1} = \text{mem}_t \cup \{\text{Reflect}(\tau_t, r_t)\}
$$

where $\tau_t$ is the trajectory of trial $t$ and $\text{Reflect}$ is the LLM itself prompted to produce a short verbal lesson:

$$
\text{Reflect}: (\tau, r) \mapsto \ell \in \Sigma^*
$$

The "gradient" is a string. The "optimizer" is in-context learning.

## How it works

Reflexion factors the agent into three LLM roles. The **Actor** $M_a$ produces actions and observations, generating a trajectory $\tau_t = (s_0, a_0, o_0, \dots, s_T)$. The **Evaluator** $M_e$ scores the trajectory, yielding a reward $r_t$ — this can be an environment signal (unit tests pass/fail), a heuristic (exact match), or another LLM call. The **Self-Reflection** model $M_{sr}$ takes $(\tau_t, r_t)$ and emits a compact natural-language post-mortem $\ell_t$: *"I assumed the list was sorted but the input contained duplicates; next time check len(set(x)) first."*

That reflection $\ell_t$ is appended to a bounded memory buffer (typically the last 1–3 reflections to fit context). On trial $t+1$, the Actor is re-prompted with the same task plus the memory, and the loop repeats until $r_t$ exceeds a threshold or a max-trials budget is hit. Crucially, the long horizon trajectory is *compressed* into a short lesson — this is what makes it scale beyond a single context window. You can think of each reflection as a single step of a very high-variance, very high-bandwidth policy gradient: one trial, one verbal "update".

## Code

```python
from openai import OpenAI
client = OpenAI()

def call(system, user):
    r = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role":"system","content":system},
                  {"role":"user","content":user}])
    return r.choices[0].message.content

def reflexion(task, tests, max_trials=4):
    memory = []
    for trial in range(max_trials):
        mem_str = "\n".join(f"- {m}" for m in memory) or "(none)"
        code = call("You are a Python coder. Return only code.",
                    f"Task: {task}\nPast lessons:\n{mem_str}")
        try:
            exec_globals = {}
            exec(code, exec_globals)
            tests(exec_globals); return code           # success
        except Exception as e:
            lesson = call("You diagnose bugs in one sentence.",
                          f"Code:\n{code}\nError: {e}\nLesson:")
            memory.append(lesson)                      # verbal "gradient"
    return None
```

## Why it matters

Reflexion's importance is less about the specific algorithm and more about what it proved: **in-context learning is expressive enough to serve as the entire RL loop** for a frontier model. Compared to RLHF/PPO it requires no reward model, no rollouts at scale, no GPU-hours of training; compared to plain Chain-of-Thought it adds the missing ingredient of *cross-trial credit assignment*. Compared to vanilla self-consistency or best-of-N, it actually *learns from failure* rather than resampling blindly.

Modern coding agents use Reflexion-shaped loops constantly: run tests → read stderr → revise. Tool-use agents use it to recover from API errors. Research agents use it to refine search queries. Even reasoning models like o1-style systems can be viewed as having internalized a Reflexion-like critique step into a single forward pass. The broader lesson — *let the model write its own lessons in English* — remains one of the highest-leverage tricks in agent design.

<Callout type="tip">
The reflection should be **shorter than the trajectory**. If your lesson buffer grows monotonically you've built a logger, not a learner. The compression step (long trajectory → one sentence) is where the implicit credit assignment happens, and it's the part most reimplementations get wrong.
</Callout>
