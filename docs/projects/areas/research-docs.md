# Research and Docs

Owns long-form thinking, specs, audits, process documents, and design research.

Primary responsibilities:

- product specs
- implementation plans
- process docs
- design audits
- project maps
- external research notes
- long-term memory of product decisions

Key folders:

- `docs/`
- `docs/process/`
- `docs/superpowers/`
- `docs/design/`
- `docs/projects/`

Design rule:

Documents should turn decisions into durable constraints, not become loose inspiration.
