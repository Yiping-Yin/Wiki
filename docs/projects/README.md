# LOOM Projects

Status: active project classification
Purpose: classify LOOM work by product area, technical ownership, and design direction.

This directory is the project-management layer for the LOOM workspace. It should describe what each area owns, where the source files live, what is active, and what should wait.

## Directory Structure

- `areas/` - durable ownership maps by product or technical area.
- `active/` - current workstreams that are being implemented or reviewed.
- `backlog/` - important future work that is not active yet.
- `archive/` - completed, superseded, or historical project notes.

## Filing Rule

Use one active note per workstream. If a task touches multiple source folders, classify it by the user-visible outcome first, then note the technical folders it affects.
